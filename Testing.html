Testing

Optimisation

After using the Performance Insight in Developer Tools in Chrome, there were some tasks that were considered as long tasks and some pictures that took longer to load than the rest. Some of the fixes for these issues are simplifying my code so that the browser is able to render code faster. As for images, I should use the correct image formats such as .png and .jpeg.

Debugging

By using the html and css validators, it was able to point out many errors in my code such as missing closing tags, errors in my element subtree, stray tags and missing semicolons in css. Some problems were solved as I had overlooked these small mistakes.

Browser testing

There were some differences between when viewing my website in the Responsive Design Mode in Chrome and FireFox. One difference was that the page in chrome has a lot of white space towards the end of the page but there was none in FireFox. Another difference was that the display my Membership Page was not the in Chrome and in FireFox. FireFox is also able to activte the :hover pseudoclass even in Responsive Design Mode while Chrom was unable to.

WAVE was able to point out many contrast errors and missing alt tags for picture which helped to improve the accessibility of the page. It pointed out at elements that had bad contrast with the background which would have been difficult for people with visual impairment to read. Its output was very clear as it was able to identify the elements that had a problem specifically, allowing me to change it immediately without spending much time looking for the element. The after screenshots show that the pages had met all the accessibility specifications. Its output was also clear in explaning the problem that had to be fixed which made it easier to use.
