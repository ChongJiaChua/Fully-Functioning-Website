Design

Introduction

The main idea for the design of this website is aiming to create a calm ambiance that allows people to feel welcome when they open up this website. The choice of cool pastel colour contributes to the design goal as it sets a peaceful tone. One of the aims of using a variety of transitions and animations is to make the page feel more interactive and stand out from other normal webpages. Since the targetted audiences are mainly students and staff, it would be better that the website makes them feel welcomed.

Site map

The home page is connected to the Purpose, Meeting & Papers, Membership and Contact pages.From there, there are different catagories under each page such as forms, quiz, lists and more.

Design reasoning

Home Page: For the home page, there are two breakpoints which supports the width of tablets and mobile phones. The breakpoint allows the grid to be in a singular column instead of two which is able to fit smaller screens. The picutres are displayed on top of each other rather than side by side because at smaller widths, the pictures will start to overlap. Tweakpoints can be use to adjust the alignment of the title inside of the grid box as the width of the screen changes. It may also be used to adjust the padding or margin to ensure that the elements are in the place desired.

Purpose Page: For the purpose page, there is only one breakpoint. Below this breakpoint, the original layout is not maintained as the width of the screen is not able to display all the elements as intended. So, the change made is for the elements that are in the same line to be arranged vertically. Tweakpoints are used to adjust the margin and padding of elements that have exceeded the screen width and to modify their positioning.

Membership Page: This page only has one breakpoint and the difference is the value of the left and right margin. By changing the margin, this allows all the content to be displayed without overflowing. Tweakpoint is used to adjust the padding within each grid cell and to adjust the gap between the grid cells.

Meetings & Papers Page: For this page, there are two breakpoints. For the narrow breakpoint, the margin around the grid is removed so that there is more space to display content when the screen becomes smaller. For the mobile breakpoint, it is similar to the other pages whereby there is only 1 grid column so that each element is able to occupy space up to the screen's width.

Contact Page: This page is similar to the rest of the pages as it uses the same grid concept. Tweakpoint is once again used to change the margin and padding so that all the content in the desired position.

Menu system

When the screen size shrinks below a certain threshold, the navigation elements will be hidden and a menu icon will appear. The memu icon utilises the checkbox input to reveal the navigation elements when interacted with. This allows for a neat display as the navigation element will not overlap each other as the screen width is too small. For the top navigation bar, the menu icon appears on the far right while for the bottom navigation bar, the menu icon appears in the middle.
