Accessibility

Accessibility considerations

1)Special characters such as the & character may not load for some devices. The &amp; was used instead to ensure all devices can load the character.
2)Images were given alt to ensure that in the event that it is not able to be displayed, a short description of the picture is given.
3) Use of ARIA roles and labels to make clear the role of different elements in the page. Link
5) Changing the color of visited links to a different color to remind users.
6) Use a variety of semantic tags to simplify the structure of the code. Link
7) Adding alternative fonts to the font-family tag in the case where the font does not display;
8) Use of light cool pastel colors instead of sharp colors so that the page content is more readable.
9) Elements that allows interaction changes its font-sizes, color, font-weight or position when the focus is on it to improve visible focus style.

Issues

Visual impairment can be solved by screen reader such as JAWS and NVDA.

Errors and contrast issues can be identified using the WAVE tool and the WebAIM contrast checker.
